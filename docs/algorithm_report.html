1. 개요

1.1 목적

스마트 WMS 갠트리 로봇 창고 시스템의 워크 핸들링 알고리즘을 최적화하여, 동일한 조건(미션 리스트)에서 갠트리 로봇의 총 이동거리를 최소화한다.

1.2 대상 시스템

항목사양
창고 규격20 x 20 x 5 (열 x 행 x 높이)
최대 적재량1,996개
입출고구 위치원점 (0, 0, 0)
적재 영역(1,1,1) ~ (20,20,5)
품목 종류4종 (01, 02, 03, 04)
적재 방식LIFO 스택 (같은 열에 수직 적재)

1.3 평가 기준

지표가중치설명
시간 점수70%총 이동거리 기반 (낮을수록 우수)
위치 점수30%적재 높이 균일성 (표준편차 낮을수록 우수)
종합 점수100%시간점수 x 0.7 + 위치점수 x 0.3

2. As-Is 분석: FF (First-Fit) 알고리즘

2.1 입고 위치 결정

입고 요청 → 가장 낮은 높이 탐색 → 좌표 순서대로 첫 번째 빈자리 선택
  • 모든 품목이 동일한 기준으로 배치
  • 출고 빈도, 출고구와의 거리 고려 없음
  • 좌표 (0,0) → (1,0) → ... → (19,19) 순서로 채움

2.2 출고 대상 선택

출고 요청 → 해당 품목의 등록순 첫 번째 lot 선택
  • 스택 내 위치(깊이), 출고구와의 거리 고려 없음
  • 위에 다른 아이템이 쌓여 있어도 그대로 선택 → 블로킹 재배치 발생

2.3 블로킹 아이템 재배치

블로킹 아이템 → 가장 낮은 높이의 첫 번째 빈자리로 이동

재배치 시에도 빈도 고려 없음

2.4 문제점

  1. 출고 빈도가 높은 품목이 창고 안쪽 깊숙이 배치될 수 있음
  2. 출고 시 블로킹 아이템이 많아 불필요한 이동 발생
  3. 모든 품목이 고르게 퍼져 평균 이동거리가 큼

3. To-Be 설계: 빈도 기반 배치 알고리즘

3.1 핵심 아이디어

출고 빈도가 높은 품목은 접근이 빠른 위치에, 낮은 품목은 깊숙한 위치에 배치한다.
XY 평면 (출고구 기준 거리)
출고구 (0,0) ← 가까움      멀리 →
HOT
COLD
색이 진할수록 접근 비용 낮음
Z축 (적재 높이)
z=5
HOT
COLD

위쪽 = 빠른 접근 (LIFO)

3.2 비용 함수

cost(x, y, z) = XY거리 + 높이패널티
              = sqrt((x + ox)² + (y + oy)²) + (HEIGHT - 1 - z)

ox, oy = 적재 영역 원점 오프셋 (1, 1)
HEIGHT = 최대 적재 높이 (5)

HOT 품목 (freq ≥ 0.5) → cost 최소 위치 선택
COLD 품목 (freq < 0.5) → cost 최대 위치 선택

3.3 입고 위치 결정 비교

As-Is (FF)

순서 기반 배치

  • 가장 낮은 높이 빈자리
  • 좌표 순서대로 선택
  • 품목 종류/빈도 무관
To-Be (RL/LA)

빈도 기반 배치

  • 품목별 출고 빈도 점수 계산
  • HOT → 위쪽 + 출고구 가까이
  • COLD → 아래쪽 + 출고구 멀리

3.4 출고 대상 선택 비교

As-Is (FF)

등록순 선택

  • 해당 품목의 첫 번째 lot
  • 블로킹/거리 고려 없음
  • 평균 2~3개 블로킹 재배치
To-Be (RL/LA)

최적 lot 선택

  • 1순위: 블로킹 수 최소 (0 우선)
  • 2순위: 출고구 XY 거리 최소
  • 불필요한 재배치 대폭 감소

3.5 블로킹 아이템 재배치 비교

As-Is (FF)

무작위 재배치

가장 낮은 높이의 첫 번째 빈자리로 이동. 빈도 고려 없음.

To-Be (RL/LA)

빈도 기반 재배치

해당 아이템의 출고 빈도에 맞는 최적 위치로 이동. 재배치 = 창고 정리.

4. 빈도 파악 방식: RL vs LA

RL

Real-time Learning (실시간 학습)

freq_score = 해당 품목 출고횟수 / 최대 출고횟수
데이터운영 중 누적 출고 이력
초기 상태0.5 (중립, FF와 유사)
학습출고 쌓일수록 정확
장점실제 환경 적용 가능, 패턴 변화 적응
단점초반 데이터 부족
LA

Lookahead (미리보기)

freq_score = 사전분석 출고횟수 / 최대 출고횟수
데이터미션 리스트 사전 분석
초기 상태처음부터 정확한 빈도
학습불필요 (사전 완성)
장점이론적 최적, 첫 미션부터 최적
단점미션 리스트 사전 필요 (테스트 전용)

6. 분석

6.1 XY 이동거리 85% 감소의 원인

FF에서는 모든 품목이 좌표 순서대로 배치되어 평균 XY 거리가 약 15 단위였다. RL/LA에서는 출고 빈도가 높은 품목이 출고구 인근(0~3 범위)에 집중 배치되어 평균 XY 거리가 약 2~3 단위로 감소하였다.

6.2 스마트 출고 선택의 효과

FF에서는 등록순 첫 번째 lot을 선택하여 평균 2~3개의 블로킹 아이템 재배치가 발생했다. RL/LA에서는 스택 최상단(블로킹 0) + 출고구 근접 lot을 우선 선택하여 블로킹 재배치가 크게 감소하였다.

6.3 RL과 LA의 성능 비교

SEED=123456: RL과 LA가 거의 동일 (이동거리 차이 0.01%)
SEED=12345: RL이 종합 점수에서 우수 (297.80% vs 270.70%) - 높이 균일성에서 RL 우위

RL은 운영 중 학습하며 배치를 점진적으로 최적화하기 때문에, 사전 분석보다 실제 패턴에 더 잘 적응하는 경향이 있다.

6.4 처리 속도

RL/LA는 FF 대비 약 20~30% 더 느리다 (위치 스코어링 연산 추가). 그러나 실제 RoboDK 시뮬레이션 모드(S)에서는 로봇 동작 시간이 지배적이므로 알고리즘 연산 시간 차이는 무시할 수 있다.

7. 결론 및 권장사항

7.1 결론

알고리즘적용 환경이동거리 감소권장
RL (실시간 학습) 실제 운영 환경 73~74% 권장
LA (미리보기) 테스트/벤치마크 73~74% 테스트용
FF (First-Fit) 기본/비교 기준 - 기준값

7.2 권장사항

1. 실제 운영에는 RL 모드 권장
   • 미션 리스트 사전 정보 불필요
   • 운영하면서 자동으로 최적화됨
   • 출고 패턴 변화에 실시간 적응
2. 벤치마크/평가에는 LA 모드 활용
   • 이론적 최적 배치로 알고리즘 상한선 확인 가능
3. 향후 개선 방향
   • 시간 감쇠(time decay): 최근 출고 이력에 가중치 부여
   • 품목별 구역 예약: 특정 품목 전용 구역 지정
   • 배치 재정렬(defrag): 유휴 시간에 기존 배치 최적화
